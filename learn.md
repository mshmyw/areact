# TDD
测试驱动开发
  方法：在开发功能代码之前，先编写单元测试代码，测试通过，则功能完成
好处：
- 主流：尤其使用框架层代码开发
- 与解决问题思路相同：测试代码就像prd,梳理你的思路
- 高质量：能够保证功能代码完备性
- 心流：全程vscode中完成，无任务切换
